Overview

Build Node Explorer from an empty repo: a two-pane, independently-navigable terminal file manager with full CRUD, in-app view (F3) with syntax highlighting, edit-in-vim (F4), and a custom-script user menu (F2). Runtime is Node ≥ 24 running .ts sources directly via native type stripping — no tsc build, no JSX (Ink is driven through htm). The work is split into four vertical slices; each ends in a runnable, testable state.

Acceptance Criteria

  • Two independently-navigable panes; Tab switches the active pane; arrows move the cursor; Enter descends into a dir and .. ascends; each pane keeps its own cwd/cursor/scrollOffset.
  • Entries sorted dirs-first, then files, alphabetical, with a leading .. row except at the filesystem root.
  • Directories show <DIR>; pressing s computes recursive size on demand and shows it (cached on the entry).
  • F7/n mkdir; F5/c copy (default dest = other pane's cwd); F6/m move; F8/d delete; r rename in place. Delete and overwrite are confirmed via a Dialog first.
  • FS-operation results and errors appear on a transient status message line, cleared on the next keypress.
  • F3/v opens an in-app scrollable read-only viewer with syntax highlighting; Esc closes; large/binary files fall back to plain text.
  • F4/e opens the file in $EDITOR (else vim) full-screen; on quit, control returns to the same pane/cursor and the file's size refreshes. (Primary acceptance test.)
  • F2/u opens the custom-script menu from config; missing/malformed config opens an empty menu with a hint; selecting an entry runs it with cwd = active pane and NE_CWD/NE_SELECTED/NE_SELECTED_PATH env, then a Press any key to continue pause, then returns and refreshes. (Secondary acceptance test.)
  • F10/q quits cleanly (raw mode restored). Both F-keys and letter shortcuts trigger the same actions.
  • node --test passes for format/fkeys/fs/ops/menu; npx tsc --noEmit reports zero errors and confirms all syntax is erasable.

Key Decisions

  1. F3 = in-app viewer using cli-highlight, windowed. Highlight the whole file once, split('\n') the ANSI output, slice the visible window per scroll (same center-clamp as FileList).
    Why: composes with the existing scroll windowing; highlighting the full file first keeps multi-line constructs (block comments/strings) correct. Impact: adds cli-highlight dep, an extension→language map, and a large/binary plain-text fallback.
  2. Dev-only typecheck from Phase 1. tsconfig.json with erasableSyntaxOnly + verbatimModuleSyntax + allowImportingTsExtensions + noEmit, plus npm run typecheck.
    Why: guarantees Node can run every .ts source untouched and catches non-erasable syntax early. Impact: typescript devDep; never used to build or run.
  3. Transient status message line in the footer for operation results/errors.
    Why: non-blocking feedback — success stays quiet, errors surface without a modal. Impact: App holds a status string cleared on next keypress.
Corrections to the source design (verified against Ink v5.2.1 source):
  • F-key sequences are ESC-stripped by Ink. Ink removes the leading \x1b before calling useInput, so fkeys.ts must map the stripped forms: F1–F4 arrive as 'OP' 'OQ' 'OR' 'OS' (SS3), F5–F10 as '[15~' '[17~' '[18~' '[19~' '[20~' '[21~' (CSI), each with key.meta === true. (The source doc's \x1b[15~ is the terminal wire form, not what Ink delivers.)
  • runInTerminal must also toggle bracketed-paste (\x1b[?2004l before spawn, \x1b[?2004h after) in addition to setRawMode(false) + stdin.pause(), or pasted input inside vim/scripts leaks \x1b[200~/\x1b[201~ artifacts.

Tech Stack & Dependencies

  • Runtime: Node ≥ 24 (native type stripping, on by default). Confirmed installed: v24.18.0.
  • Runtime deps: ink ^5react ^18.3htm ^3cli-highlight ^2.1 (no react-dom — Ink uses its own reconciler).
  • Dev dep: typescript ^5 for tsc --noEmit only.
  • No JSX, no tsc for running. Files are .ts, rendered with const html = htm.bind(React.createElement).

Phase 1 — Scaffold & Read-Only Two-Pane Navigation

Goal: A runnable two-pane explorer. Both panes list a directory, sort correctly, scroll, and navigate independently with Tab / arrows / Enter / ... Establishes the htm+Ink pattern, type-stripping config, F-key decoding, and the status line — all wired but only navigation actions active.

Tasks

  • package.json: "type":"module"; deps ink@^5, react@^18.3, htm@^3; devDep typescript@^5; scripts start (node src/index.ts), test (node --test), typecheck (tsc --noEmit); bin → src/index.ts; engines.node ≥ 24. Run npm install.
  • tsconfig.json: noEmit, erasableSyntaxOnly, allowImportingTsExtensions, verbatimModuleSyntax, module/moduleResolution = nodenext, target es2023, jsx not set, strict.
  • src/types.ts: Entry ({ name, isDir, size, mtime, sizeComputed? }), PaneState ({ cwd, entries, cursor, scrollOffset }), Modal union, MenuItem. Export via export type only (no runtime body).
  • src/util/format.ts: formatSize(bytes) (human units), formatDate(mtime), and a column/padding helper for the listing row (name / size / mtime).
  • src/util/fkeys.ts: decodeFKey(input, key) mapping ESC-stripped sequences → ids only when key.meta === true (so a bare [15~ in file content / paste is not misread as F5). F1–F4 = OP/OQ/OR/OS; F5–F10 = [15~ [17~ [18~ [19~ [20~ [21~. Only the bound keys need actions (F2/F3/F4/F5/F6/F7/F8/F10); F1/F9 may be omitted or decoded-but-unbound. Returns null for non-F-key input.
  • src/core/fs.ts: listDir(cwd) → fs.readdir(cwd,{withFileTypes:true}) + per-entry fs.lstat for size/mtime (skip symlink targets); sort dirs-first, then files, alphabetical (locale-aware, case-insensitive); prepend .. unless cwd is the fs root. Directory size starts unknown → rendered <DIR>.
  • src/ui/FileList.ts: windowFor(cursor,total,height) center-clamp windowing (per disk-reclaim); render visible slice, remap index via start+i; highlight cursor row (inverse); dim the inactive pane's cursor.
  • src/ui/Pane.ts: path header (truncate-middle) + FileList + footer; receives active flag.
  • src/ui/StatusBar.ts: F-key hint bar plus a status message line prop (last action/error; empty by default).
  • src/ui/App.ts: two PaneState, active: 0|1, modal (none for now), status string. useInput routes: Tab toggles active; arrows/PgUp/PgDn move cursor (clamped); Enter on dir → chdir (reset cursor), on .. → parent, on file → no-op stub; F10/q → useApp().exit(). Any keypress clears status. viewHeight = max(3, (process.stdout.rows||24) - chrome).
  • src/index.ts: #!/usr/bin/env node; render(html`<${App}/>`); initial cwd = process.cwd() for both panes.
  • README.md: project summary, install/run, keybindings table (mark F2/F3/F4/menu as later phases).
  • Tests: test/format.test.js (size/date formatting, padding), test/fkeys.test.js (each F-key sequence + key.meta → id; a bare [15~ without key.meta → null; non-F-key → null), test/fs.test.js (sorting order dirs-first/alpha, .. presence and root suppression against a temp dir).

Phase 2 — File Operations, Dialogs & Directory Size

Goal: Full CRUD from the UI with confirmation/input dialogs, Norton-Commander dest defaults, error surfacing on the status line, and on-demand recursive directory sizing.

Tasks

  • src/core/ops.ts (node:fs/promises): copy(src,dest) → fs.cp(src,dest,{recursive:true}); move(src,dest) → fs.rename with EXDEV cross-device fallback to cp+rm; remove(path) → fs.rm(path,{recursive:true,force:true}); makeDir(path) → fs.mkdir; rename(src,dest) → fs.rename. Each returns a result/throws a typed error for the status line. Add an exists(path) helper for overwrite detection.
  • src/core/dirSize.ts: recursive lstat walk summing file sizes (skip symlinks); returns total bytes. Result cached on the Entry (size + sizeComputed=true).
  • src/ui/Dialog.ts: two modes — confirm (message + Yes/No, Enter=confirm, Esc/N=cancel) and text-input (label + editable field, Enter=submit, Esc=cancel). Driven by modal state; routes input while open.
  • Wire actions in App.ts (F-key + letter → same handler):
    • F7/n → input-mkdir dialog → makeDir in active pane; refresh.
    • F5/c → confirm-copy (default dest = other pane's cwd, shown) → copy; if dest exists, escalate to overwrite confirm; refresh both panes.
    • F6/m → confirm-move (dest = other pane) → move; overwrite confirm as above; refresh both.
    • F8/d → confirm-delete → remove; refresh; clamp cursor.
    • r → input-rename (prefilled with current name) → rename in place; refresh; keep cursor on renamed entry.
    • s → dirSize on the hovered <DIR>; replace <DIR> with the formatted size (cached).
  • Success/error → status line (e.g. ✓ Copied report.pdf → /home/other / Permission denied); guard against operating on the .. row.
  • Tests: test/ops.test.js (temp-dir round-trips: mkdir → copy → rename → move → delete; cross-device fallback path exercised or documented as skipped), plus a dirSize case (known tree → expected total).
  • README: fill in the operations keybindings and dest-default behavior.

Automated Verification

  • node --test passes including ops + dirSize.
  • npx tsc --noEmit reports zero errors.

Manual Verification milestone

  • F7/n creates a folder; it appears in the list.
  • F5/c on a file copies to the other pane (default dest shown); the file appears there.
  • F6/m moves; F8/d deletes after confirm; r renames in place.
  • Copying/moving onto an existing name triggers an overwrite confirm.
  • s on a <DIR> row replaces <DIR> with the recursive size.
  • A failing op (e.g. delete a protected path) shows the error on the status line without crashing.

Phase 3 — Shell Integration: Edit-in-Vim (F4) & In-App Viewer (F3)

Goal: The delicate TTY hand-off. runInTerminal suspends Ink, hands the terminal to a child process, and resumes cleanly. F4 edits in vim; F3 opens the in-app highlighted read-only viewer.

Tasks

  • Add dep cli-highlight@^2.1; npm install.
  • src/ui/useShell.ts: runInTerminal(cmdOrArgs, opts) using useStdin() → { setRawMode, isRawModeSupported, stdin }:
    • If !isRawModeSupported → graceful no-op (return early).
    • setRawMode(false); stdin.pause?.(); write \x1b[?2004l (disable bracketed paste).
    • spawnSync(...) with stdio:'inherit' (vim: spawnSync(editor,[absPath]); script: spawnSync(cmd,{shell:true,cwd,env})).
    • If opts.pause: write Press any key to continue… and read one keypress (fs.readSync(0, …)) before restoring — while stdin is still paused/raw-off (Phase 4 uses this).
    • Restore (last): write \x1b[?2004h; stdin.resume?.(); setRawMode(true) — so Ink recaptures stdin only after any pause read.
  • src/util/lang.ts: extension → highlight.js language id map, validated via getLanguage(id); returns undefined (auto/none) for unknown.
  • src/ui/Viewer.ts: read the file; if size > threshold (~1 MB) or binary (NUL-byte sniff) → plain text; else highlight(code,{language}) once, split('\n') → ANSI lines. Windowed scroll (reuse windowFor): ↑/↓ line, PgUp/PgDn page, Esc closes. Header shows file name + size; each visible line rendered as <Text wrap="truncate-end">.
  • Wire F4/e in App.ts: on a file → runInTerminal($EDITOR||'vim', absPath); on return, refresh the active pane (size may have changed) and keep pane/cursor. No-op on a dir / ...
  • Wire F3/v in App.ts: open Viewer modal on a file; ignore on a dir / ...
  • README: document F3 (viewer + highlighting + fallback) and F4 ($EDITOR) behavior.

Automated Verification

  • npx tsc --noEmit reports zero errors.
  • node --test still passes (optional: a lang.ts ext→id unit test; the TTY runInTerminal path is verified manually).

Manual Verification milestone — primary acceptance

  • F4/e on a file → vim opens full-screen; on :q control returns to Node Explorer at the same pane/cursor; the file's size refreshes. No leaked escape sequences (background-color query / bracketed paste) after return.
  • F3/v on a source file → highlighted, scrollable read-only view; ↑/↓ and PgUp/PgDn scroll; Esc closes to the same pane/cursor.
  • F3 on a large or binary file → plain-text fallback, no crash, no garbled colors.
  • Ctrl+C still exits cleanly during/after a spawn (Ink's raw-mode handling intact).

Phase 4 — Custom-Script User Menu (F2)

Goal: An F2 user menu of configured scripts that run in the terminal (reusing runInTerminal) with the active pane as cwd and the hovered entry exposed via env, followed by a Press any key to continue pause.

Tasks

  • src/core/menu.ts: loadMenu() resolves the config path — $NODE_EXPLORER_MENU if set, else ~/.config/node-explorer/menu.json; parse JSON; validate each { key?, label, command }; drop invalid entries. Missing or malformed file → return empty list (with a reason) rather than throwing.
  • src/ui/MenuDialog.ts: overlay listing entries (label + optional hotkey); choose via arrows+Enter or by pressing an entry's key; Esc cancels. Empty menu shows a hint pointing at the config path.
  • Wire F2/u in App.ts: open MenuDialog; on select, run via runInTerminal(item.command, { shell:true, cwd: activePane.cwd, env: { ...process.env, NE_CWD, NE_SELECTED, NE_SELECTED_PATH }, pause:true }).
  • Press any key to continue… pause: after the child exits, write the prompt, read one keypress (fs.readSync(0, …)) before resuming Ink, then refresh the active pane (the script may have changed the dir).
  • Tests: test/menu.test.js — valid config parses to entries; malformed JSON → empty + reason; missing file → empty + reason; invalid entries filtered.
  • README: document the menu, the config-path resolution order, the JSON shape, and the NE_* env vars; include an example menu.json.

Automated Verification

  • node --test passes including menu (valid / malformed / missing / invalid-entry cases).
  • npx tsc --noEmit reports zero errors.

Manual Verification milestone — secondary acceptance

  • With a configured menu.json, F2/u lists the entries; arrows+Enter and per-entry hotkeys both select.
  • Selecting e.g. Git status runs it in the terminal with cwd = active pane, shows Press any key to continue…, then returns to the same pane/cursor with the listing refreshed.
  • A script that reads $NE_SELECTED/$NE_SELECTED_PATH/$NE_CWD sees the correct hovered entry and directory.
  • Missing/malformed menu.json → menu opens empty with a hint; no crash.

Cross-Cutting Notes

  • Type-stripping rules (enforced by tsconfig, verified by tsc --noEmit): explicit .ts extensions on local imports; import type for type-only imports; no enums, runtime namespaces, parameter properties, or declare fields.
  • Both key schemes (F-keys via fkeys.ts, letters v e c m n d r q s u read from input) call the same action handlers. When a modal/viewer/menu is open, input routes there instead of pane navigation.
  • Tests are .js importing .ts sources — Node strips types on import; run with node --test.
  • Refresh discipline: after any op / edit / script, re-run listDir for affected pane(s) and clamp the cursor.

Out of Scope for v1 (possible follow-ups)

Multi-file marking (Insert key), archive browsing, built-in text editor, filesystem watching/auto-refresh, custom keybinding config. Menu follow-ups: an other-pane cwd env var, inline macro placeholders (e.g. %f), a per-project menu file. Viewer follow-ups: theme selection, line numbers, search.
